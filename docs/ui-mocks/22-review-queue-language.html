Review queue language concept

One review state model, reused across queue, tree, and file view.

The inspector should not explain work through a separate next-action card. The useful part is the three-state summary and the queue itself. Rows inside a state section do not need to repeat that state. Mixed surfaces, such as the file tree and the open file header, can reuse the same quiet state label.

Queued Review candidates waiting for a human decision. Unread remains only a blue dot.
In Review Files with open threads or active review conversation.
Reviewed Completed review work. If a new HEAD change appears, the file moves back to Queued automatically.

Same label in mixed surfaces

The state label is useful where files are mixed together. It can be quiet: a small label with a colored dot, reused in the tree and file header.

Same slot in the file view

The open file keeps one review-state slot in the header. The value changes between Queued, In Review, and Reviewed. The completion action appears only when the current state can become Reviewed.
