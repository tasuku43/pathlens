Review queue inspector refinement

Five flatter versions of Pattern A, without blocks inside blocks.

These keep the same state model and work order as Pattern A, but remove the feeling that every group and every file is a separate card. The right sidebar should read like one continuous worklist.

A Hairline Groups B State Rail C Header Bands D Count Gutter E Inline States
Keep A Queued, In Review, and Reviewed remain the primary mental model.
Flatten rows File rows use separators, active tint, and text weight instead of card borders.
Quiet history Reviewed is recoverable but visually below the active work.
Action home Mark as reviewed stays in the file view header, while the inspector stays a worklist.

Pattern A

Hairline Groups

The closest evolution from the current A. Section headers remain, but the section body is just a continuous list separated by hairlines.

  • Most conservative replacement.
  • Still clearly grouped and collapsible.
  • Removes the card-inside-card feeling.

Pattern B

State Rail

A single vertical rail carries the grouping. The list no longer needs boxes because the colored spine communicates state changes.

  • Feels modern and spatial.
  • Good when unread dots and status color matter.
  • Tradeoff: a little more custom visual language.

Pattern C

Header Bands

The group header becomes the only block-like surface. File rows are table rows beneath it, so the hierarchy has one level of emphasis.

  • Very legible at right-inspector width.
  • Section labels stay strong without cards.
  • Tradeoff: a little more enterprise-like.

Pattern D

Count Gutter

Counts move into a narrow left gutter. This removes repeated summary cards and lets the main column behave like one continuous list.

  • Strongest reduction of block count.
  • Counts stay visible next to their rows.
  • Tradeoff: less conventional than plain sections.

Pattern E

Inline States

The flattest version. There are no group containers; state is a small row column, and ordering still keeps Queued first.

  • Cleanest visual surface.
  • Works well with future sorting or filtering.
  • Tradeoff: weaker sense of section boundaries.
